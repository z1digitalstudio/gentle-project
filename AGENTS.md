# Project Instructions

## Contexto del proyecto

Este repo es un template para equipos de producto. Tiene un proceso documentado para Discovery, Design y Development que el agente debe seguir.

## Antes de trabajar

Lee [ai-playbook/README.md](/ai-playbook/README.md) para entender qué fases existen y cómo está estructurado el proceso.

## Cómo operar

- Antes de arrancar una fase, consultá el README de esa fase en `ai-playbook/guides/`
- Antes de ejecutar una skill, consultá su doc en `ai-playbook/guides/<fase>/skills-docs/`
- Seguí los criterios de cierre documentados — no des una subfase por cerrada si no se cumplen
- Los artefactos canónicos viven en `docs/`, `product/` y `management/` — respetá esos paths

## Lo que no debe pasar

- No improvisar el proceso cuando hay documentación que lo define
- No crear artefactos fuera de los paths canónicos sin razón explícita
- No saltear subfases sin verificar que los inputs de la siguiente están disponibles
