# Gentle Project Template

Plantilla para equipos de producto que quieren ejecutar Discovery, Design y Development con una lógica clara, asistencia de IA y artefactos documentados.

Está pensada para proyectos digitales en los que un equipo humano — PMs, diseñadores y desarrolladores — trabaja junto a un agente de IA para estructurar el proceso desde la investigación hasta el desarrollo.

## Para quién es esto

*Product Managers*, *Product Designers* y *Developers* que quieren un punto de partida estructurado para trabajar con IA en proyectos de producto — sin improvisar el proceso en cada proyecto nuevo.

## Qué incluye

```
ai-playbook/     Guías para el equipo humano: fases, subfases, skills y artefactos
skills/          Skills locales del repo (ejecutables por el agente de IA)
docs/            Documentos canónicos del proyecto (PROJECT-SUMMARY.md, etc.)
product/         Artefactos de producto (RESEARCH.md, PRD.md, USER-STORIES.md)
management/      Artefactos de gestión (ROADMAP.md, RISKS.md, DECISIONS.md)
design/          Artefactos de diseño
```

## Requisitos

Un agente de IA compatible: Claude Code, Codex, Pi u OpenCode.

## Cómo empezar

1. Usa este repositorio como plantilla para tu proyecto (botón "Use this template" en GitHub)
2. Abre el proyecto con tu agente de IA
3. Lee el [AI Playbook](/ai-playbook/README.md) — explica las fases, cuándo usar cada skill y qué artefactos producen
4. Empieza por [Discovery](/ai-playbook/guides/discovery/README.md)
